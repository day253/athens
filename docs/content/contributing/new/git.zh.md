# Git 贡献指南

## 分支策略
- `main`: 稳定的生产环境代码
- `develop`: 功能集成分支
- `feature/*`: 功能分支 (例如 `feature/user-auth`)
- `bugfix/*`: 错误修复分支

## 提交信息格式
```
<类型>(<范围>): <主题>

<正文>

<页脚>
```

### 类型:
- feat: 新功能
- fix: 错误修复
- docs: 文档变更
- style: 代码格式化
- refactor: 代码重构
- test: 添加测试
- chore: 维护任务

### 示例:
```
feat(用户): 添加登录功能

- 添加了邮箱/密码认证
- 实现了 JWT 令牌生成

关闭 #123
```

## 拉取请求流程
1. Fork 代码库
2. 创建功能分支
3. 提交更改
4. 推送到你的 Fork
5. 向 `develop` 分支发起拉取请求
